- Questo applicativo è stato realizzato seguendo un approccio iterativo, in particolare nelle prime iterazioni ci si è concentrati
a implementare le funzionalità più importanti individuate nei requisiti. A conclusione di tale lavoro e dopo aver applicato i 
test unitari con JUnit (con relative correzioni) è stata prodotta la versione del programma "BookBoutique-v0.4.1.jar" presente 
in questa cartella. 

- Il file "BookBoutique-v0.5.jar" invece rappresenta l'ultima versione implementata comprensiva di accesso ad Database e delle 
ultime funzionalità rimaste in sospeso.

- Si osservi che la funzione di invio delle email per le prenotazioni completate funziona realmente. Inoltre il database remoto
è hostato sulla piattaforma Heroku.

- La cartella "documentation" contiene tutta la documentazione realizzata, in particolare la cartella "doc-completa" contiene
i file di documentazione definitivi (che le sono già stati inviati via email). Le altre cartelle ("01-Ideazione", "02-Elaborazione1", ecc)
contengono i documenti prodotti via via che si procedeva con la progettazione e con l'implementazione dell'applicativo. Infine il
file DomainModel.asta contiene i modelli di Dominio prodotti nelle varie iterazioni; i diagrammi di sequenza di sistema (SSD) invece
sono stati realizzati con lo strumento Drow.io Pro offerto da Google Drive e dunque sono presenti solo i file immagine.

- La cartella "src" contiene il codice sorgente dell'applicazione, mentre "lib" contiene le librerie esterne utilizzate (javax.mail.jar, 
postgresql-9.4-1201.jdbc4.jar, hibernate).
